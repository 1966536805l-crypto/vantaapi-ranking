# Tool Page Content Template

Use this template for VantaAPI tool pages. Keep it concise and useful.

## Hero

```text
{Tool Name}
{One-line practical promise}
```

Example:

```text
Prompt Optimizer
Turn a rough request into a clearer prompt for writing, coding, or learning.
```

## What this tool does

```text
This tool helps you {primary action}. It is best for {main use case}.
```

## Who it is for

- Beginners who need structure.
- Independent developers preparing tasks or docs.
- Learners who want clearer examples and next steps.

## Input example

```text
I want to build a simple landing page for a GitHub repo analyzer.
```

## Output example

```text
Role: You are a frontend developer.
Task: Build a focused landing page for a GitHub repository analyzer.
Requirements: include URL input, loading state, analysis sections, and limitations.
Acceptance criteria: responsive layout, clear empty/error states, no backend changes.
```

## FAQ

### Does this replace expert review?

No. It helps structure your work, but you should still review important output before using it.

### Does it need private tokens?

No. MVP tools should avoid asking for private tokens unless a dedicated authenticated flow exists.

## Usage limits

- Do not paste secrets.
- Do not paste private customer data.
- Review generated output before using it in production.
- Treat tool output as a draft, not a guarantee.
